This assignment moves beyond simple happiness metrics to construct a robust Quality of Life (QoL) analysis. You will investigate the intersection of Economic Wealth, Personal Safety (Crime), and Individual Liberty using a composite database.

Accept Assignment

In the repository lives a file named project_qol.sql. You should download this file. It represents a database dump. The easiest, and fastest, method is to use the command line tool psql. 

psql -d analysis -U yourusername -f project_qol.sql

This will create three tables: happiness, freedom, and countries. Ensure you identify the primary keys (Country Names/Codes) that link them together.

Problem 1: The Hierarchy of Needs

Before a population can focus on "Happiness," they must first be safe. We want to define a "Livable" country as one that meets a baseline safety threshold.

Part A

Using the freedom table, identify the top 5 countries with the highest Security & Safety score (pf_ss). Display the Country Name, Region, and Safety Score.

Part B

Does money buy safety? JOIN the countries table with the freedom table. Group the data by Region. Calculate the average GDP per Capita and the average Safety Score (pf_ss). Order by Safety Score descending.

Part C

Find the "Hidden Gems." Write a query that returns countries with a pf_ss score above the global average, but a GDP per capita below the global average. Return Country, GDP, and Safety Score.
